文章の方向
writing-mode: キーワード値;


writing-mode は、テキストの行のレイアウトを横書きにするか縦書きにするか、 ブロックのフロー方向を左向きにするか右向きにするかを設定する CSSプロパティ です。


horizontal-tb 横書き、上から下へ改行するキーワード値
vertical-lr 縦書き、右から左へ改行するキーワード値
vertical-rl 縦書き、左から右へ改行するキーワード値


~~~

書字の方向
direction: キーワード値;


ltr 書字方向を左書き(左から右)に指定するキーワード値
rtl 書字方向を右書き(右から左)に指定するキーワード値


ブロック内の書字方向は、CSSプロパティ direction で指定できますが、 ページの構造に大きく関わるため、CSSではなく グローバル属性 dir を使用するべきとされています。


dir="ltr" lang="ja">このブロックの書字方向は左書き(左から右)です。


dir="rtl" lang="ja">このブロックの書字方向は右書き(右から左)です。


dir="auto" lang="ja">このブロックの書字方向は言語に基づき自動設定されます。



lang属性
<要素名 lang="ja">


書字方向は、言語によって崩れる場合があります。 rtl を アラビア語などの 右書きの言語 以外に設定する必要はほとんどありません。


~~~


direction: rtl;
unicode-bidi: bidi-override;


unicode-bidi は、direction プロパティと共に、文書内の双方向テキストの扱いを指定する CSSプロパティです。
値を bidi-override に設定すると、言語を問わず direction の方向に テキストが表示されます。

縦書きモード専用のプロパティ
text-orientation: キーワード値;
text-combine-upright: キーワード値;


text-orientation と text-combine-upright は、 writing-mode の値が vertical-rl または vertical-lr の場合のみ に効果がある CSSプロパティです。


~~~


text-orientation

mixed 横書き用の文字を右に 90° 回転させ、縦書き用の文字は自然に配置するキーワード値
upright 横書き用の文字を、縦書き用の字形と同様に自然に (正立で) 配置するキーワード値
sideways 全行を横書きで書いて 90° 回転したように配置するキーワード値
sideways-right sideways の別名です。


~~~


text-combine-upright は、縦書きモードのブロック内のインラインテキストの書字方向を指定する CSSプロパティです。

none 特別な処理をしないキーワード値（初期値）
all すべての文字を水平に並べてボックス内に収めるキーワード値
digits 指定した桁数までの連続した ASCII 数字(半角数字) を水平に並べてボックス内に収めるキーワード値

digits 値に 対応しているブラウザーはあまり多くありません。
