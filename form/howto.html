Form-associated Content

フォームとは、ウェブサイトやアプリケーションとユーザーとの対話の要となる仕組みです。

フォームは 1つ以上のコントロール（ウィジェットともいいます）と、フォーム全体を構成するのに役立つ関連要素で作られます。

action="URL" method="HTTP メソッド">
すべてのフォームは、form 要素から始まります。

フォームの動作方法は、action 属性と method 属性で設定します。

action="データを送信すべき場所 (URL)"
action属性の値は、妥当な相対/絶対 URL でなければなりません。
action属性がなければ、フォームが含まれているページの URL にデータが送信されます。

method="データを送信するために使用するHTTP メソッド"
get: フォームのデータは action 属性の URL に ? で区切って追加されます。
post: フォームのデータはリクエスト本文として送信されます。
dialog: フォームが dialog 要素の中にある場合に使用し、ダイアログが閉じて submit イベントを発行しますが、データを送信したりフォームをクリアしたりはしません。

GET リクエストはユーザーが URL バーの中でデータを見ることができますが、POST リクエストではそうではありません。
パスワードを送信する必要がある場合は、GET メソッドを使用してはいけません。
大量のデータを送信する必要があるなら、POST が好ましいメソッドです。

target="閲覧コンテキスト" _self: 現在と同じ閲覧コンテキスト
_blank: 無名の新しい閲覧コンテキスト
_parent: 現在の親の閲覧コンテキスト
_top: 上位の閲覧コンテキスト
for="ID名"">

label を input 要素と関連付けると、いくらかの利点が発生します。

label を input に関連付けるには、input に id属性を設定し、id と 同じ値を label の for属性の値に設定します。
複数のラベルに同じfor値を設定し、同じinput に 複数の label を関連付けることもできます。

type="button" の input 要素には label を関連づけるべきではありません。
label の中にアンカーやボタンのような対話型要素を配置しないでください。
label の中に見出し要素（h1 ~ h6）を配置すると、支援技術を妨げることになります。

type="キーワード値" name="コントロール名"

input 要素は、HTML の中で最も強力かつ最も複雑な要素の一つです。

フォームを構成するコントロール（ウィジェット）のほとんどは、input 要素を使って作成されます。

input 要素において、もっとも重要な属性は type 属性です。
input 要素に type 属性を設定し、ユーザーがあらゆる種類のデータを入力できるウィジェットをフォームに追加します。

name 属性を使って コントロール（ウィジェット）に名前を設定します。
フォームは、input 要素 の name 属性に設定した名前と入力値(valueの値)の組を送信します。

input 要素にはコンテンツを含むことができないため、終了タグを省略することができます。

JavaScript では HTMLInputElement.value プロパティを使って値を受け取ることができます。

type="text"
基本的な単一行の入力欄です。改行は自動的に入力値から取り除かれます。

type="email"
正しい形式のメールアドレス（username@domain）のみを受け取る入力欄です。
multiple 属性が指定されたとき（のみ）、値はカンマ区切りで複数の正しい形式のメールアドレスを入れることができます。
CSS の :valid および :invalid 擬似クラスが自動的に適用され、フィールド上の現在の値が妥当なメールアドレスであるかどうかを視覚的に示します。

type="url"
正しい形式の URL（urlscheme://restofurl）のみを受け取る入力欄です。
CSS の :valid および :invalid 擬似クラスが自動的に適用され、フィールド上の現在の値が妥当なURLであるかどうかを視覚的に示します。

type="tel"
電話番号のための入力欄です。
入力される電話番号の長さの最大値は maxlength 属性で設定することができます。
特定のパターンに適合させたい場合は、pattern属性を使用してください。

type="password"
テキストが読み取られることがないように、文字がアスタリスク ("*") やドット ("•") のような記号に置き換えられる入力欄です。
パスワードを自動的に入力するため、ユーザーのパスワードマネージャを許可するには、autocomplete 属性を指定します。
minlength 属性や maxlength 属性を使って、許容する最小の長さと最大の長さをパスワードに適用することができます。
pattern 属性を使って、パスワードがそれらの要件に合致しているか自動的に確認するための正規表現を設定することができます。

入力欄に共通する属性

required
入力欄への入力を必須に設定します。

placeholder="テキスト"
入力欄が空欄の時に表示するテキストを設定します。

spellcheck="false"
スペルチェックを有効にします。

autocorrect="on"
打ち間違いの自動修正や、テキストの置き換え処理を有効にします。

チェック可能項目
チェックボックスはグループ内のボタンの複数にチェックができ、 ラジオボタンはグループ内のボタンのうち同時に一つしかチェックできません。

チェック項目のラベル（キャプション付け）には label 要素を使用し、 label要素のfor属性の値 と input要素のid属性の値 に共通の名前を設定します。

for="ラベル名"> type="checkbox" name="グループ名" id="ラベル名" value="チェックボックスの値"
for="ラベル名"> type="radio" name="グループ名" id="ラベル名" value="ラジオボタンの値"

同一のグループに属するチェック項目の name 属性に同じ値(グループ名)を設定します。
fieldset で 同一グループの項目をまとめると便利です。

value 属性に、項目が選択されたときに送信する値を設定します。
checked 属性が設定された項目は、ページが読み込まれたときに自動的に選択されます。

/* ~~~ */

value="値">

section は、選択式のメニューを提供するコントロールを表します。
multiple 属性がついている場合は、複数の項目を選択できます。
size 属性で、選択肢をその行だけ表示できるようにを設定できます。

それぞれの選択肢は、section の中に option 要素を記述して作成します。
option の 開始タグと終了タグの間に記述したテキストが選択肢として表示し、 valueの値をフォームから送信します。
disabled 属性をした選択肢は、選択不能になります。
selected 属性をした選択肢は、初期状態で選択されます。

select 要素の選択肢 option を optgroup 要素で囲み、選択肢のグループを制作できます。
optgroup 要素 に label属性を設定すると、その値がグループ名としてコントロールに表示されます。

name="テキストエリア名">

メッセージの入力などに使用する複数行のテキストフィールドです。

既定値を定義したい場合は、開始タグと終了タグの間にテキストを記述します。
テキストエリアが空欄の時に表示するテキストは、placeholder 属性で設定できます。

label 要素と結びつける場合は、id属性を設定します。
minlength と maxlength 属性を使用して、文字数を設定できます。
disabled 属性を設定した要素はどのような方法でも選択できません。 (値の送信もされません)
readonly 属性を設定した要素は選択可能で内容をコピーできますが、内容を編集できません。 (値は送信されます)

type="ボタンの種類">

クリックできるボタンを表し、フォームや文書で単純なボタン機能が必要なあらゆる場所で使用することができます。
ボタンには 3つの種類があり、input 要素にも 同じ種類の type があります。

type="submit">
type="submit"
submit ボタンをクリックすると、フォームのデータを action 属性で定義したウェブページへ method 属性で定義した方法で送信します。

type="reset">
type="reset"
reset ボタンをクリックすると、フォーム内の入力欄すべてを初期値にリセットします。
通常、フォームにリセットボタンを配置するべきではありません。このようなボタンは便利なものではなく、間違ってボタンをクリックしてしまったユーザーをイライラさせる可能性が高くなります。

type="button">
type="button"
クリックしても何の操作も行わない、単純なボタンです。
独自のボタンを作成するのに役立ち、JavaScriptでその機能を定義することができます。

フィールドセットのキャプション

fieldset 要素は、スタイルや意味付けのために 同じ目的を持つウィジェットをグループ化するために使います。

fieldset 要素の初めの小要素の legend 要素は、グループ化したフィールドセットにキャプションを提供します。
