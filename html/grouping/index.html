Grouping Content

グループ化要素（グルーピング・コンテンツ）

HTML Living Standard (HTML5以降) から新たに追加された メイン・ヘッダー・フッター要素 は、ページの基本となる構造を表します。

文章をベースとした記事ページには、区分要素（セクショニング・コンテンツ） が適切です。

<main>メイン要素</main>

<main> 要素 は、主要な内容 (中心的な文章・機能) を表します。
文書には hidden 属性が指定されていない メイン要素 を 複数使用できません。

hidden属性 が設定された メイン要素は ブラウザに表示されません。
hidden>非表示状態のメイン要素

<header>ヘッダー要素</header>

<header> 要素 は、導入的なコンテンツ (ロゴ・検索フォーム・ナビゲーション等) を表します。
header 要素は通常、自身を囲む section の見出し (h1–h6 要素) を含むことを意図していますが、必須ではありません。

<footer>フッター要素</footer>

<footer> 要素 は、フッター (連絡先情報・関連文書へのリンク・著作権情報等) を表します。
セクションに対してフッターを使用し、コンテンツの公開日やライセンス情報など記述することにも使用されます。

従来のHTML要素 (HTML 4.01)

HTML Living Standard 以前の HTML では、<div> と <span> を使いすぎる傾向があったため、 メイン・ヘッダー・フッター要素やセクション要素などの意味的要素が追加されました。

<div>コンテンツ区分要素</div>

<div> 要素 は、ブロックレベルの汎用コンテナーを表します。
適切な意味的要素（main・header・footer・article・nav など）がない場合に使用します。

<span>汎用的インライン要素</span>

<span> 要素 は、インラインレベルの行内コンテナーを表します。
スタイル付けのため (class または id 属性を使用して) 要素をグループ化する用途で使用します。

HTML Living Standard では、 <div> と <span> は意味のないコンテンツでのみ使用し、意味を持つコンテンツには適切な意味的要素を使うことが推奨されています。
