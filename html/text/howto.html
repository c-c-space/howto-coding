強調と重要性
<em> と <strong>

em と strong は、段落の中で特に重要な部分をマークアップする場合に使用する セマンティック要素 （テキストレベルセマンティックス） です。
このマークアップは、視覚的に文書を読みやすくするだけでなく、読み上げソフトの音声が異なるトーンで発音する効果も期待できます。

<em> 強調要素 </em>

周囲の文字列と比較して強調される言葉 に em 要素を使用します。
通常、em 要素は斜体で表示されますが、CSS で書体を変更できます。
単に斜体のスタイルを適用させたい場面では、i 要素が適切かもしれません。

<strong> 強い重要性要素 </strong>

内容の重要性、重大性、または緊急性が高いテキスト に strong 要素を使用します。
通常、strong 要素は太字で表示されますが、CSS で フォントの太さ (あるいは重み) を変更できます。
単に太字のスタイルを適用させたい場面では、b 要素が適切かもしれません。


~~~

CSS
font-style: 通常体 (normal)、筆記体 (italic)、斜体 (oblique);
font-weight: フォントの太さ (あるいは重み) ;

書体・太字の指定は 以上の CSS プロパティで指定するべきです。
テキストのスタイリングに セマンティック要素 を使うのは適切ではありません。

×

イタリック、太字、下線
<i> <b> <u>

CSS がまだ十分に対応されていないなかった時代に、太字・斜体・下線付きのテキストを書くために B I U 要素は作られました。
アクセシビリティ、SEO などが重要になった現在では、これらの要素はもはや使用するべきではないとされています。


<i> 慣用的なテキスト要素 </i>

何らかの理由で他のテキストと区別されるテキストの範囲を表します。
英文において、慣用句・技術用語・分類学上の呼称などが よくイタリック体で表現されていたことが i という名前の元になっています。

<b> 注目付け要素 </b>

要素の内容に読み手の注意を惹きたい場合で、他の特別な重要性が与えられないものに使用します。
以前は太字要素と呼ばれており、ほとんどのブラウザーで文字列が太字で表示されます。

<u> 非言語的注釈 (下線) 要素 </b>

非言語的に注釈があることを示す方法で表示する行内テキストの区間を示します。
以前は "Underline" (下線) 要素と呼ばれる傾向があり、現在でもそのように誤用されることがあります。


~~~

CSS
font-style: 通常体 (normal)、筆記体 (italic)、斜体 (oblique);
font-weight: フォントの太さ (あるいは重み) ;
text-decoration: テキストの装飾的な線;

書体・太字・下線の指定は 以上の CSS プロパティで指定するべきです。
テキストのスタイリングに セマンティック要素 を使うのは適切ではありません。

×

小さめのテキスト
<small> <sup> <sub>

著作権表示や法的表記のような注釈、表記規則上の理由などで、文字を小さく表示する必要がある場合に使用する、いくつかの セマンティック要素 （テキストレベルセマンティックス） があります。
これらの要素は 表現や表示の結果を得るためではなく、表記上の理由で、テキストの位置（サイズ）を変更する必要がある場合でのみ使用するべきです。

<small> 附随コメント要素 </small>
既定では、 small から x-small のように、一段階小さいフォントサイズでテキストが表示されます。

<sup> 上付き文字要素 </sup>
<sub> 下付き文字要素 </sub>
表記上の理由で、上付き・下付き文字として表示するべき文字列に使用します。


~~~

CSS
font-size: フォントの大きさ;
vertical-align: 垂直方向の配置を設定;

フォントサイズ・上付き・下付き文字の指定は 以上の CSS プロパティで指定するべきです。
テキストのスタイリングに セマンティック要素 を使うのは適切ではありません。

インライン要素
<span>

記述コンテンツに、汎用的なインラインブロックを作成する際に使用します。
スタイル付けのために、class または id 属性を使用して、要素をグループ化する用途でよく使用されます。

改行（改行可能位置）と区切り
<br> <wbr> <hr>

段落内で文章を改行を生成する際に、br 要素を使用します。
br に 終了タグは必要ありません。

wbr 要素は、テキスト内でブラウザーが任意で改行してよい位置を表します。
ウインドウ内にテキスト（文節）が収まっている場合など、改行が必要ない場合は改行を行いません。

段落と段落とで話題が切り替わる場合、hr 要素を使用して 段落を区分します。
以前は水平線として定義されていて、現在でもブラウザーには水平線が表示されます。


~~~

CSS
border: 境界の値;
padding: パディングの値;
margin: マージンの値;

段落間の間隔の調整や水平線のスタイリングには 以上の CSS プロパティを使用します。
段落間の間隔を広げるために空の段落や区切り要素を乱用するのは適切ではありません。
