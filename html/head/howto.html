文書メタデータ (ヘッダー) 要素

文書メタデータ (ヘッダー) 要素 は、 HTML 1.0 から使われる最古の要素の一つで HTMLの 最初の要素 として、一つだけ 配置できます。

HEAD要素 は、ページが読み込まれても ウェブブラウザには表示されない 部分ですが、HTMLについてよい基礎を与える ページタイトル・ メタデータ・ ファビコン などの情報を設定する重要な要素です。

コンピューターは、HEAD要素の情報を元に、ブラウザにページを表示します。
HEAD要素のはじめには、ページがあらゆるOS・言語に対応できるよう 2つ の メタデータコンテンツ を設定するのが一般的です。 

<meta charset="utf-8">
<meta name="viewport" content="width=device-width, initial-scale=1.0">

utf-8 は普遍的な文字セットで、この設定は あらゆる人間の言語のほとんどすべての文字を扱うことができることを意味します。
viewport は ブラウザの表示領域を設定するメタデータで、この設定は 端末の画面サイズや方向を認識し、ページを最適化することを意味します。

この記述は、W3C・WHATWG(Apple, Google, Mozilla, Microsoft)が推奨するcharset属性とviewportの設定方法です。


format-detection は モバイル端末に向けた指示を設定するメタデータです。 

<meta name="format-detection" content="telephone=no">

telephone=no は、数字の羅列を電話番号と認識する自動リンク機能を無効化する設定です。 

<meta name="format-detection" content="address=no, email=no">

content属性に、address=no, email=no を含めると、メールアドレスや住所への自動リンク機能が無効化されます。
※ Android端末の一部では、メールアドレスや住所が自動的にリンクされてしまう場合があるそうです。

※ これはモバイル端末向けの機能で、HTML Living Standardでは正式に定義されていません。


次に、ブラウザのタイトルバーやページのタブ、ブックマーク・検索エンジンなどに提供する情報を設定します。 

<title> ページタイトル </title>
<meta name="description" content="ページのディスクリプション（説明書き）">
<meta name="author" content="著者">
<meta name="reply-to" content="Eメールアドレス（連絡先）">

TITLE要素は、 HTML 1.0 から使われる最古の要素の一つで コンピューター・検索エンジンなどに、ページタイトルを定義します。
ページタイトルは、検索エンジンのアルゴリズムが検索結果にページを列挙する順序を決定するために使用する材料の一つです。
一般的に、長く説明的な題名は、短く一般的な題名よりも効果があります。（検索エンジンは通常、ページタイトルとしておよそ 55–60 文字を表示します。）

コンピューター・検索エンジンなどは、ページタイトルに加えて、ページの著者 (author)・ページの説明 (description) などの内容から ページを評価します。

説明 (description) に ページの内容に関連したキーワードをうまく取り入れるなど メタデータを充実させ、検索エンジンでページをより上位にするために対策することを Search Engine Optimization（検索エンジン最適化）または SEOといいます。

メタデータ要素

META要素は HTML 2.0 から使われている要素で、 さまざまなブラウザ・OS、コンピョーターにとって重要な情報を設定します。

現在では、 特定のサイト（SNS サイトなど）が特定の情報の断片を取得するために独占的に設計されたメタデータ など、さまざまな型があります。


外部リソースへのリンク

LINK要素 は、 HTML 1.0 から使われる最古の要素の一つで カスタムアイコン・HTMLに適用する CSSなど、 現在の文書に関係する外部のリソースを指定します。
リンク要素のリンク種別属性(rel)に 外部リソースの種類を設定し、 href属性に外部リソースの場所を設定します。

rel="icon" href="カスタムアイコン"

もっとサイトデザインを豊かにするために ファビコン・Webクリップアイコン（ホーム画面のタッチアイコン）など のカスタムアイコンをHTMLにリンクします。

カスタムアイコンのサイズは、コンピョーターによって様々です。
一般的なカスタムアイコンのサイズ一覧は 外部リソースへのリンク要素 を参照ください。


/* スタイル情報要素 */

まだCSSが充実していなかったころは、HEAD要素内の STYLE要素に CSS を 記述していました。
現在は、CSSを外部スタイルシートに記述して、LINK 要素を使用して HTML に適用する方法が推奨されています。

rel="stylesheet" href="styles.css"


文書の基底URL要素

<base href="基底URL" target="閲覧コンテキスト">

BASE要素は、 HTML 1.0 から使われる最古の要素の一つで 文書内におけるすべての相対URL の基点となる URL を指定します。
BASE要素は、HEAD内に 1つ だけ置くことができ、URL の属性値を持つ LINK要素 などの前に置く必要があります。

href
相対 URL に使用される基底 URLを設定する属性
絶対 URL と相対 URL が使用できます。

target
a または form 要素が target 属性を持たない場合に移動する閲覧コンテキストを設定する属性
閲覧コンテキストについては アンカー要素とID属性 を参照ください。
