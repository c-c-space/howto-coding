Event
ウェブサイトに対話性を追加するには、イベントが必要です。

イベントは、ブラウザーの中で起きていることを検出し、その応答としてコードを実行するコード構造です。

最も分かりやすい例は、 要素をクリックしたときに実行する click イベント です。

EventTarget.addEventListener('click', event => {
// JavaScriptを記述
});

EventTarget.onclick = function() {
// JavaScriptを記述
}

/* ~~~ */

EventTarget.addEventListener ('イベント名', 関数);
イベントハンドラー・イベントリスナー

イベントには様々な型があり、 addEventListener() を使って DOM 要素に ベントハンドラー（イベント名と実行する文） を設定します。

MouseEvent
mousemove – マウスが移動したときに実行
mousedown / mouseup – マウスボタンを押した/離したときに実行
mouseenter / mouseleave – マウスカーソルが要素へ来た/出たときに実行

TouchEvent
touchmove – タッチ感応面で点が移動したときに実行
touchstart / touchend – タッチ感応面に点が配置/取り除かれたときに実行
touchcancel – タッチポイントが実装固有の方法で中断されたときに実行

KeyboardEvent
keydown – キーが押された状態のときに実行
keyup – キーが離されたときに実行

Scroll & WheelEvent
scroll – スクロールされたときに実行
wheel – マウスホイールを動かしたときに実行

readystatechange
文書の読み込み状態によってイベントを実行

loading
文書の読み込み中に実行する

interactive
文書の読み込みは完了したが、CSS・画像・動画などのサブリソースがまだ読み込み中の状態で実行する

complete
文書とすべてのサブリソースの読み込みが完了した状態で実行する

  if (event.target.readyState === 'loading') {
    // 文書の読み込み中に実行する
  }

  else if (event.target.readyState === 'interactive') {
    // window.addEventListener('DOMContentLoaded', event => { DOMContentLoaded イベントと同じ状態 })
  }

  else if (event.target.readyState === 'complete') {
    // window.addEventListener('load', event => { load イベントと同じ状態 })
  }

});
